Session Report — Manipulation Backfire Model

Branch: appnet  |  Date: 2026-04-30  |  Author: Tomoya Takeda

Overview

This session redesigned the experimental targeting strategy, corrected network-generation directedness across all topology implementations, added a new directed scale-free network model (DMS), and built a full post-hoc structural analysis pipeline. The core research question is: does the structural position of a targeted moderate influencer modulate the magnitude of the backfire effect?

1. Targeting Strategy Redesign

modified src/admin/ExpIntervention.java

Problem

The previous getManipulationTarget always selected the single highest-follower moderate agent (opinion < 0.2) across all seeds. Because structural properties (betweenness, clustering, degree) co-vary strongly with follower rank, the old approach conflated "influencer type" with "largest hub in this seed's network." Cross-seed comparison was confounded.

Change

Two-stage selection is now applied:

  1. Pool construction — all agents satisfying |opinion| < 0.2 and followers ≥ ⌈0.05 × N⌉ (= 50 for N=1000) are collected. The follower floor prevents structurally marginal agents from being targeted.
  2. Uniform random draw — one agent is drawn uniformly at random from the pool. This decouples targeting from structural dominance, creating natural variation in target structural type across seeds.
int followerFloor = (int) Math.ceil(0.05 * n);
// collect pool, then:
int chosen = pool.get(randomGenerator.get().nextInt(pool.size()));

Runs where the pool is empty (no moderate agent meets the floor) are skipped with a [WARN] log line. Pool size is logged per run to monitor regime shifts during long multi-seed batches.

Experimental implication: run 50+ seeds (both ±1 target direction) to generate sufficient structural diversity for post-hoc correlation analysis.

3. Directed Barabási–Albert Network

modified src/network/BarabasiAlbertNetwork.java

What changed and why

The previous directed version tracked only in-degree in the degree pool. New nodes entered with kin = 0 and were therefore never selectable by later arrivals (zero attachment probability), breaking the preferential-attachment dynamic. The degree distribution was near-degenerate, dominated by seed-clique nodes.

The fix uses total-degree (in + out) for preferential attachment, exactly as in the original undirected BA model. When newNode → target:

  • newNode is added to the pool (its out-degree just increased → total degree++)
  • target is added to the pool (its in-degree just increased → total degree++)

This recovers γ = 3 for the total-degree distribution. New nodes are in the pool from birth (weight = m from their out-edges), so late-arriving nodes can still accumulate followers.

4. New Network: DMS Directed Scale-Free

new src/network/DMSNetwork.java

Implements the Dorogovtsev–Mendes–Samukhin (2000) directed scale-free model. Attachment probability is proportional to kin(i) + A, where A > 0 is the initial attractiveness parameter.

  • In-degree distribution: P(kin) ~ k−(2 + A/m)
  • Out-degree: exactly m for all post-seed nodes (not scale-free)
  • New node reachability: guaranteed by A > 0 — all nodes have non-zero base weight even at kin = 0

Compared to BarabasiAlbertNetwork, DMS is the cleaner formulation for a follower-graph because only follower count (in-degree) drives hub formation — out-degree plays no role in attracting new followers.

5. Structural Backfire Analysis Pipeline

new structural-backfire-analysis.ipynb

A self-contained analysis notebook that connects the structural position of each run's targeted hub to the magnitude of the backfire effect observed in that run.

Pipeline

  1. Target identification — reads step_20000.gexf per run directory; locates the node with target=True via a namespace-stripped XML parser.
  2. Structural metrics — computed on the step-20000 network snapshot:
    MetricInterpretationNotes
    In-degreefollower count at intervention timedirect reach
    Betweenness centralitybridge / broker positionnormalised, exact
    Local clustering coefficientecho-chamber density around targeton undirected projection
    PageRank (α=0.85)prestige via random-walkrobust on disconnected graphs
    Eigenvector centralityhub score (left eigenvector)NaN fallback on non-convergence
  3. Backfire effect — scalar per run: 
    backfire = (Δopposite_share) − (Δtarget_share)
         where Δ = mean(steps 39000–40000) − mean(steps 19000–20000)
    Positive value = opposite-direction posting grew more than target-direction posting.
  4. Scatter plots — one panel per metric (5 total); Spearman ρ with significance stars; OLS trend line; points coloured by target direction (±1).
  5. Output — results/summary/structural_backfire.png and structural_backfire_table.csv.

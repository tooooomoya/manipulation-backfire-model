Re-anchoring Stylized Fact 1 (Apparent Polarization) to Empirical Posting-Rate Data

Technical report — derivation, data, and reproduction

Manuscript Manipulating Moderate Hubs in Social Networks Section 3.2 / SF1 Date 2026-06-11 Author T. Takeda

This report documents why the empirical anchor for SF1 (apparent polarization) was changed, how the new target [1.19, 1.26] was computed from raw account-level data, and how to reproduce it. It is meant to make the chain of reasoning fully auditable after the fact.

1. The problem with the original anchor

The submitted draft operationalized SF1 as the ratio of observable post-content variance to underlying opinion variance,

R = σ2post / σ2opinion

and validated the model's value (1.232) against a target of [1.10, 1.25], citing Eady et al. (2019) and Barberá et al. (2015). Auditing those citations revealed a construct mismatch:

What the citation actually measuresWhat SF1 needs
Eady (2019): standard deviation of the ideology of accounts a user is exposed to — 0.87 (via retweets) vs. 0.78 (direct follows). This is a network-exposure quantity about which accounts are seen. Variance of the content a user posts relative to that user's own latent opinion — a within-/between-author production quantity.
Verdict: Eady's 0.87/0.78 measures exposure composition, not post-vs-opinion dispersion. The [1.10, 1.25] target had no traceable source in either cited paper — the 0.87/0.78 = 1.12 → 1.12² ≈ 1.25 conversion in the validation notebook silently equated two different constructs. NOT DEFENSIBLE

2. The correct construct: posting rate by ideological extremity

The model represents an information space sliced by a single salient issue (a "hashtag" space). On such a slice, SF1 is generated by one mechanism: users with more extreme opinions post more often, so the distribution of posts over opinion is more dispersed than the distribution of users over opinion. The right empirical comparand is therefore a posting-rate-by-ideology measurement, not an exposure measurement.

We found a direct, public source for exactly this:

Buntain & Snegovaya (2025), "Post-January 6 deplatforming shows long-term effects on ideological polarization among Twitter users", PNAS Nexus 4(11), pgaf333.
Reported (Fig. 2): hyper-partisan accounts ("Very Liberal" / "Very Conservative") post about twice as often as moderates.
Public data: OSF 10.17605/OSF.IO/KQFM6 — anonymized account-level daily post counts (_anon_data/twitter.2020to2021.matched.counts.parquet) joined to self-reported ideological lean (tw_handle_to_libcon.csv).

3. Method

Let each surveyed user have a self-reported lean x ∈ {−3,…,+3} (7 categories). Each post is assumed to carry its author's lean. Then:

  • σ2opinion = variance of x weighted by the number of users at each position (the opinion-holder distribution).
  • σ2post = variance of x weighted by the number of posts contributed at each position (the observable-content distribution).
  • R = σ2post / σ2opinion.

The ratio is invariant to any affine rescaling of the opinion axis, so the specific [−3,+3] coding does not affect R; it assumes only that the seven categories are approximately equally spaced.

6. Reproduction

Download the two OSF files and run:

# OSF 10.17605/OSF.IO/KQFM6  (_anon_data)
#   twitter.2020to2021.matched.counts.parquet  -> tw_counts.parquet
#   tw_handle_to_libcon.csv                     -> tw_libcon.csv

import pandas as pd, numpy as np

counts = pd.read_parquet('tw_counts.parquet')          # 762 x 737 daily post counts
lc = pd.read_csv('tw_libcon.csv').set_index('handle')  # handle -> ideological_lean [-3,+3]
counts = counts.loc[lc.index]
x = lc['ideological_lean'].values

def wvar(x, w):
    w = np.asarray(w, float); x = np.asarray(x, float)
    m = np.sum(w*x)/np.sum(w)
    return np.sum(w*(x-m)**2)/np.sum(w)

def ratio_for(cols):
    posts = counts[cols].sum(axis=1).values   # posts per account
    s2_op   = wvar(x, np.ones_like(posts))    # user-weighted  (opinion holders)
    s2_post = wvar(x, posts)                  # post-weighted  (observable content)
    return s2_post / s2_op

allcols = list(counts.columns)
print("full    :", ratio_for(allcols))                                   # 1.192
print("election:", ratio_for([c for c in allcols if c[:7] in ('2020-10','2020-11')]))  # 1.257
print("jan2021 :", ratio_for([c for c in allcols if '2021-01' in c]))    # 1.195

7. Caveats

  • Scale invariance / equal spacing. R is unaffected by the [−3,+3] coding but assumes the 7 categories are equally spaced.
  • Proxy for latent opinion. Self-reported lean (discrete, 7-point) stands in for the continuous latent opinion the model uses.
  • Between-author only. The empirical ratio reflects only that extremes post more; it does not capture any within-author content amplification.
  • Single, non-representative sample. The matched sample leans liberal and is not a population-representative panel (authors' own caveat). The band is an order-of-magnitude reference, not a population parameter.
  • "~2×" is descriptive. Buntain & Snegovaya's regression shows extremity has a negative baseline association with activity that turns positive only via an event-driven time trend — i.e. the gradient is concentrated in high-salience windows, consistent with our use.
